### 1. Analisis Kode dan Solusi Pemecahan Masalah

```javascript
function processUser(user) {
  if (!isValidCandidate(user)) return null;
  return doSomething(user);
}

function isValidCandidate(user) {
  return user && user.isActive && user.hasPermission;
}
```

1. Menghindari Deep Nesting (Sarang Bersusun)
Kode pertama menggunakan pola di mana statemen if bersusun bertingkat-tingkat ke dalam (sering disebut sebagai Arrow Anti-Pattern). Semakin dalam tingkat nesting-nya, semakin sulit otak kita untuk mengikuti alur logika dan mengingat konteks pada setiap tingkatan. Kode kedua berhasil "meratakan" struktur tersebut.

2. Penggunaan Guard Clauses (Early Return)
Kode kedua menerapkan teknik Guard Clause. Alih-alih mengecek kondisi "jika benar maka jalankan", ia mengecek "jika salah, segera hentikan fungsi dan kembalikan nilai (return null)". Ini membuat alur utama dari fungsi processUser sangat jelas: jika tidak valid, berhenti; jika valid, lanjutkan proses doSomething(user).

3. Pemisahan Tanggung Jawab (Separation of Concerns)
Pada kode kedua, logika untuk memvalidasi user dipisah ke dalam fungsi tersendiri yaitu isValidCandidate().

Jika suatu saat ada bug terkait perizinan atau status aktif user, kamu tahu persis harus mencarinya di dalam fungsi isValidCandidate.

Fungsi processUser menjadi sangat bersih karena ia hanya fokus pada tugas utamanya: memproses user.

4. Lebih Mudah Diuji (Testability)
Karena logika validasi dipisah menjadi isValidCandidate(), kamu bisa membuat Unit Test khusus untuk fungsi tersebut secara mandiri dengan berbagai skenario (misal: user null, user tidak aktif, user tidak punya izin) tanpa harus menjalankan keseluruhan fungsi processUser.

Secara keseluruhan, kode kedua menerapkan prinsip Clean Code yang membuat pemeliharaan aplikasi (terutama untuk jangka panjang atau saat dikerjakan dalam tim) menjadi jauh lebih efisien.

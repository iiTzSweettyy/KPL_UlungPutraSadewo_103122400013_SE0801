### 1. Analisis Kode dan Solusi Pemecahan Masalah

Berdasarkan kode awal yang diberikan, fungsi fetchOrderDetails belum merepresentasikan prinsip kode yang bersih (Clean Code) karena fungsi tersebut memborong terlalu banyak tugas sekaligus di dalam satu lingkup (scope).

Permasalahan (Bad Practice):

Pelanggaran Single Responsibility Principle (SRP): Fungsi utama secara bersamaan melakukan pengambilan data dari API, manipulasi antarmuka/DOM (membuat elemen satu per satu menggunakan document.createElement), dan mengatur visibilitas modal. Hal ini membuat fungsi menjadi monolithic, sulit dibaca, dan menyulitkan proses debugging.

Risiko Memory Leak (Bug Event Listener): Setiap kali fungsi dipanggil untuk melihat detail pesanan, aplikasi akan menambahkan addEventListener baru pada tombol penutup (Close) dan tombol konfirmasi (Confirm). Jika pengguna membuka 5 pesanan berbeda, mengklik tombol konfirmasi akan memicu fungsi confirmOrder sebanyak 5 kali secara bersamaan karena listener yang lama tidak pernah dibersihkan.

Struktur Asinkron Bersarang: Penggunaan rentetan Promise (.then()) membuat blok kode menjadi menjorok ke dalam (nested) dan kurang rapi dibandingkan pendekatan sinkron.

Solusi Penanganan (Fix):
Masalah ini diselesaikan melalui proses refactoring dengan memecah fungsi raksasa tersebut menjadi beberapa fungsi kecil modular yang berfokus pada satu tanggung jawab spesifik (Separation of Concerns).

Pemisahan Layer: Kode dibagi menjadi Data Layer (khusus API), View Layer (khusus manipulasi DOM), dan Controller (orkestrator utama).

Optimasi Manipulasi DOM: Pembuatan elemen DOM yang bertele-tele diganti dengan Template Literals yang disisipkan melalui innerHTML agar jauh lebih ringkas.

Perbaikan Logika Event Listener: Listener untuk tombol close dipindahkan ke tahap inisialisasi awal sehingga hanya dipasang satu kali. Untuk tombol konfirmasi, addEventListener diganti dengan penetapan properti langsung (onclick = ...) agar handler yang baru otomatis menimpa handler yang lama, mencegah terjadinya eksekusi ganda.

Implementasi async/await: Rantai Promise diubah menjadi async/await agar alur logika dapat dibaca secara lurus dari atas ke bawah.

  **Perbaikan pada `index.js`:**

  ```javascript
    // ==========================================
    // 1. DATA LAYER
    // ==========================================
    async function fetchOrderData(orderId, token) {
        const response = await fetch(`https://example.com/api/order/${orderId}`, {
            headers: { 'Authorization': token }
        });
        
        if (!response.ok) {
            throw new Error('Gagal mengambil detail pesanan');
        }
        return response.json();
    }

    // ==========================================
    // 2. VIEW LAYER
    // ==========================================
    function renderOrderDetails(order) {
        const detailsDiv = document.getElementById('orderDetails');
        
        // Menggunakan Template Literal untuk HTML yang lebih bersih
        detailsDiv.innerHTML = `
            <h3>Order ID: ${order.id}</h3>
            <p>Status: ${order.status}</p>
        `;
    }

    function toggleModal(isVisible) {
        const modal = document.getElementById('orderModal');
        modal.style.display = isVisible ? 'block' : 'none';
    }

    function setupConfirmButton(order, token) {
        const confirmBtn = document.getElementById('confirmOrderBtn');
        
        if (order.status === 'Delivered') {
            confirmBtn.style.display = 'none';
        } else {
            confirmBtn.style.display = 'inline-block'; // Pastikan tombol muncul jika sebelumnya disembunyikan
            
            // Menggunakan properti onclick untuk menghindari duplikasi event listener
            // setiap kali modal dibuka dengan data pesanan yang berbeda.
            confirmBtn.onclick = () => confirmOrder(order.id, token);
        }
    }

    // ==========================================
    // 3. CONTROLLER
    // ==========================================
    async function handleViewOrderDetails(orderId, token) {
        try {
            // 1. Ambil data
            const order = await fetchOrderData(orderId, token);
            
            // 2. Perbarui UI
            renderOrderDetails(order);
            setupConfirmButton(order, token);
            
            // 3. Tampilkan Modal
            toggleModal(true);
        } catch (error) {
            console.error('Error:', error.message);
            // Opsional: Tampilkan pesan error ke pengguna di sini
        }
    }

    // ==========================================
    // 4. SETUP AWAL
    // ==========================================
    function initializeModalListeners() {
        const closeBtn = document.querySelector('#orderModal .close');
        // Event listener untuk tombol close cukup dipasang SATU KALI saja
        closeBtn.addEventListener('click', () => {
            toggleModal(false);
        });
    }

    // Panggil setup ini saat file dimuat
    initializeModalListeners();
  ```
